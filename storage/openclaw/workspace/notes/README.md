# Notas del workspace

Estructura:
- `tasks/` — tareas y pendientes
- `diary/` — diario personal (un archivo por día: YYYY-MM-DD.md)
- `projects/` — notas por proyecto

## Comandos útiles para el agente
- "crea una tarea: X" → crea o actualiza `tasks/pending.md`
- "agrega al diario: X" → escribe en `diary/YYYY-MM-DD.md`
- "nota sobre proyecto X: Y" → escribe en `projects/X.md`
- "muestra mis tareas" → lee `tasks/pending.md`
